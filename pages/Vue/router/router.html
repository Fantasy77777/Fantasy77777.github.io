# vue-router

# hash模式和history模式

hash模式：(vue-router 默认 hash 模式)

url中带有#符号，hash 模式的原理是 onhashchange 事件(监测hash值变化)，可以在 window 对象上监听这个事件。

history模式：

优雅的显示完整的url，需要单独设置服务端，比如Nginx要添加这样的配置才能正常运行

location / {
  try_files $uri $uri/ /index.html
}

# 路由跳转方式

第一种 声明式

// params传参
<router-link :to="{name:xxx,params:{key:value}}">valueString</router-link>
// 获取参数
this.$route.params.key

// query传参，显示在url中
<router-link :to="{name:xxx,query:{key:value}}">valueString</router-link>
// 获取参数
this.$route.query.key

// 在路由中定义好参数，冒号后面的就是,也会在url中显示
{
  path:'/xxx/:id/',
}
<router-link to="/xxx/123456">params</router-link>

params 和 query 的区别：

params要指定路由name，不能是path，且参数不会显示在url中

query两者都可以，但是参数会显示在url中

// params
this.$router.push({
  name: 'xxx', // 只能用name来指定路由
  params: {
    key: 'value'
  }
})

//query
this.$router.push({
  // name: 'xxx', // name也可以
  path: '/xxx',
  query: {
    key: 'value'
  }
})

第二种 声明式

// param

//query
this.$router.push({name:'xxx',query:{key:value}})

this.$router.go(-1)//跳转到上一次浏览的页面

this.$router.replace('/xxx')//指定跳转的地址

this.$router.replace({name:'xxx'})//指定跳转路由的名字

push 和 replace 的区别：

使用push方法的跳转会向 history 栈添加一个新的记录，当我们点击浏览器的返回按钮时可以看到之前的页面。也就是说是支持后退的。

使用replace方法不会向 history 添加新记录，而是替换掉当前的 history 记录，即当replace跳转到的网页后，'后退' 按钮不能查看之前的页面。

# 路由导航

// 全局前置守卫：
router.beforeEach((to, from, next) => {
  // to就是跳转的路由
  // from就是当前准备离开的路由
  // next()方法是释放操作
}

// 全局后置守卫:
router.afterEach((to, from) => {
  // do something
})

// 单独路由的守卫：
const router = new VueRouter({
  routes: [
    {
      path: '/xxx',
      component: xxx,
      beforeEnter: (to, from, next) => {
        // do something
      }
    }
  ]
})
